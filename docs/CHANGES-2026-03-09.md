# Changes — 2026-03-09: Portable venv wrappers + search bug fix

Summary of all changes made to the zeresh-brain repo during the zhizi local
MCP setup session.

## 1. Bug fix: `brain-cli.py` search breaks with filters

**Affected:** `search` command when using `--project`, `--kind`, or `--since`.

**Root cause:** The SQL query has `%s` placeholders in two places — first in
the `SELECT` clause (embedding vector, ts_rank), then in the `WHERE` clause
(project/kind/since filters). psycopg2 binds params positionally through the
SQL text top-to-bottom. But `params_q` was built as
`[WHERE_params] + [SELECT_params, limit]`, so the WHERE filter value (e.g. the
string `"zhizi"`) was fed to `%s::vector`, causing:

    invalid input syntax for type vector: "zhizi"

**Fix (two lines in `brain-cli.py`):**

```python
# Semantic search path (was: params + [emb_str, query, limit])
params_q = [emb_str, query] + params + [limit]

# Keyword fallback path (was: params + [query, query, limit])
params_q = [query] + params + [query, limit]
```

SELECT-clause params now come first, matching the SQL placeholder order.

**This bug also exists on bakkies** — it just didn't surface because search was
mostly used without filters.

## 2. New: `brain` and `brain-mcp` wrapper scripts

**Problem:** Modern Debian (Python 3.12+) enforces PEP 668 — `pip install` to
the system Python is blocked. Dependencies must live in a venv. But hardcoding
the venv path in shebangs makes scripts non-portable across machines.

**Solution:** Two bash wrapper scripts that self-discover the repo location:

```bash
#!/bin/bash
SCRIPT_DIR="$(cd "$(dirname "$(readlink -f "$0")")" && pwd)"
exec "$SCRIPT_DIR/.venv/bin/python3" "$SCRIPT_DIR/brain-cli.py" "$@"
```

Same pattern for `brain-mcp` → `brain-mcp.py`.

Users symlink these to `~/.local/bin/`. Works on any machine regardless of
checkout path, as long as `.venv/` exists in the repo root.

## 3. Updated `.mcp.json` — now portable

**Before:**
```json
{ "command": "python3", "args": ["/home/oc/projects/zeresh-brain/brain-mcp.py"] }
```

**After:**
```json
{ "command": "brain-mcp" }
```

Finds `brain-mcp` via PATH. Can be symlinked into any project directory on any
machine.
